🎯 Purpose
To digitally record, manage, and visualize wildlife species data.

To support conservation planning, habitat monitoring, and research.

To integrate field survey data with spatial layers for better decision-making.

🗂️ Core Components
Database Layer (PostGIS): Stores species inventory (plants & animals), attributes, and survey records.

GeoServer: Publishes spatial data as WMS/WFS services for web access.

WebGIS Portal (Leaflet/ArcGIS Online): Interactive map interface for visualization and querying.

Survey Tools: Questionnaires for animal/plant inventory, GPS-based data collection.

📌 Key Features
Species Mapping: Plot animal and plant species with GPS coordinates.

Habitat Classification: Dense forest, grassland, riverine, dry deciduous zones.

Inventory Attributes: Species name, scientific name, category (mammal, bird, reptile, etc.), count, age group, gender.

Observation Records: Behavior, conservation concerns, human activity signs.

Interactive Queries: Filter by species, range, habitat type.

Data Export: For academic reports, conservation studies, and government records.

🌍 Benefits
Provides a centralized digital repository of biodiversity data.

Enhances real-time monitoring of species distribution.

Supports conservation strategies by identifying vulnerable zones.

Facilitates academic research and internship projects on biodiversity.
